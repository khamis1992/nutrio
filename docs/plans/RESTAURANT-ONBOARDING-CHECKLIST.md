# Restaurant Partner Onboarding Checklist

Use this checklist when onboarding each new restaurant partner to ensure consistency and completeness.

## Pre-Meeting Preparation

### Research (Before Contact)
- [ ] Review restaurant menu and pricing
- [ ] Check online reviews (Google, Instagram)
- [ ] Verify cuisine type fits our healthy focus
- [ ] Check location is within delivery zone
- [ ] Review social media presence

### Materials to Prepare
- [ ] Partnership proposal deck
- [ ] Sample partnership agreement
- [ ] Payout structure explanation
- [ ] QR code marketing materials sample
- [ ] Tablet/phone for demo
- [ ] Business cards
- [ ] Notepad for notes

---

## Initial Meeting (In Person Preferred)

### Introduction (15 minutes)
- [ ] Introduce yourself and Nutrio Fuel
- [ ] Explain the subscription marketplace concept
- [ ] Show demo of customer app
- [ ] Show demo of partner dashboard
- [ ] Share success stories from other markets
- [ ] Address initial questions

### Understanding Their Business (20 minutes)
**Ask about:**
- [ ] Current daily order volume
- [ ] Average order value
- [ ] Peak hours/days
- [ ] Current delivery setup
- [ ] Kitchen capacity (meals per day)
- [ ] Staff availability
- [ ] Current customer acquisition challenges

**Assess:**
- [ ] Food quality and presentation
- [ ] Kitchen cleanliness
- [ ] Staff professionalism
- [ ] Packaging quality
- [ ] Overall fit for platform

### Presenting the Partnership (15 minutes)
- [ ] Explain payout structure (typically 20-30 QAR per meal)
- [ ] Show revenue projection example
- [ ] Explain zero marketing cost benefit
- [ ] Describe guaranteed minimum orders (if offering)
- [ ] Explain quality standards and ratings
- [ ] Show QR code marketing concept

### Addressing Concerns (10 minutes)
**Common concerns to address:**
- [ ] "What if we get too many orders?" → Daily capacity limits
- [ ] "How do we get paid?" → Weekly automatic transfers
- [ ] "What about our existing customers?" → Can continue as normal
- [ ] "Can we leave the platform?" → Yes, with 30-day notice
- [ ] "Who handles complaints?" → Shared responsibility

---

## Application Phase

### Step 1: Basic Information
- [ ] Restaurant legal name
- [ ] Trade license number
- [ ] Owner/Manager name and contact
- [ ] Restaurant phone number
- [ ] Restaurant email address
- [ ] Website (if any)
- [ ] Instagram handle

### Step 2: Location & Operations
- [ ] Full address
- [ ] Google Maps location pinned
- [ ] Operating hours (open/close times)
- [ ] Days open/closed
- [ ] Delivery radius they're comfortable with
- [ ] Average prep time per meal
- [ ] Maximum daily capacity

### Step 3: Cuisine & Menu
- [ ] Primary cuisine type
- [ ] Dietary specialties (keto, vegan, gluten-free, etc.)
- [ ] Average calories per meal
- [ ] Price range (for reference, not displayed)
- [ ] Signature dishes
- [ ] Any items they DON'T want on platform

### Step 4: Banking Information (For Payouts)
- [ ] Bank name
- [ ] Account holder name
- [ ] IBAN number
- [ ] Account number
- [ ] SWIFT code (if international)
- [ ] Confirmation of account ownership

### Step 5: Media Collection
- [ ] Restaurant logo (high res, transparent background preferred)
- [ ] Restaurant exterior photo
- [ ] Restaurant interior photo
- [ ] Kitchen photo
- [ ] 5-10 meal photos (high quality, well-lit)
- [ ] Chef/owner photo (optional)

**Photo Requirements:**
- [ ] Minimum 1200x800 pixels
- [ ] Well-lit, professional quality
- [ ] Actual dishes (not stock photos)
- [ ] Consistent style/plating

---

## Approval Process

### Internal Review
- [ ] All information complete
- [ ] Photos meet quality standards
- [ ] Location verified on map
- [ ] Bank details confirmed
- [ ] Menu fits healthy criteria
- [ ] No red flags from research

### Set Payout Rate
- [ ] Determine per-meal payout (20-30 QAR typical)
- [ ] Consider their current pricing
- [ ] Negotiate if needed
- [ ] Document agreed rate
- [ ] Set in system

### Approval Decision
- [ ] Approve application
- [ ] Send welcome email with next steps
- [ ] OR reject with reason
- [ ] Schedule follow-up if needed

---

## Onboarding Completion

### Technical Setup (Day 1)
- [ ] Create partner account
- [ ] Set payout rate in system
- [ ] Add restaurant profile
- [ ] Upload all photos
- [ ] Set operating hours
- [ ] Set daily capacity limit
- [ ] Test login credentials

### Menu Setup (Day 2-3)
- [ ] Add 10-20 meals to platform
- [ ] Write compelling descriptions
- [ ] Add calorie counts
- [ ] Add macro information (protein/carbs/fat)
- [ ] Add dietary tags
- [ ] Upload meal photos
- [ ] Set prep times
- [ ] Mark items available/unavailable

**For Each Meal:**
- [ ] Name is clear and appealing
- [ ] Description mentions key ingredients
- [ ] Calories accurate (verified)
- [ ] Photo is appetizing
- [ ] Dietary tags correct
- [ ] Prep time realistic

### Training (Day 3-4)
- [ ] Schedule training session (30 min)
- [ ] Walk through partner dashboard
- [ ] Show how to:
  - [ ] View incoming orders
  - [ ] Mark orders as prepared
  - [ ] Update meal availability
  - [ ] View earnings and payouts
  - [ ] Update profile
- [ ] Explain order notification process
- [ ] Provide WhatsApp support number
- [ ] Share training video/recording

### Test Orders (Day 4-5)
- [ ] Place 2-3 test orders
- [ ] Verify notifications received
- [ ] Check order preparation flow
- [ ] Test "mark as ready" function
- [ ] Verify customer communication
- [ ] Get feedback from restaurant
- [ ] Fix any issues

### Go-Live (Day 6-7)
- [ ] Change status to "approved"
- [ ] Restaurant visible to customers
- [ ] Notify restaurant of go-live
- [ ] Monitor first few orders closely
- [ ] Provide extra support for first week
- [ ] Check in daily for first 3 days

---

## Post-Launch Support

### Week 1 Check-ins
- [ ] Day 1: How was first day?
- [ ] Day 3: Any issues with orders?
- [ ] Day 7: Weekly summary and feedback

### Ongoing (Monthly)
- [ ] Monthly performance report
- [ ] Review ratings and feedback
- [ ] Discuss capacity adjustments
- [ ] Update menu if needed
- [ ] Address any concerns
- [ ] Share marketing opportunities

---

## Quality Standards Checklist

### Minimum Requirements
- [ ] Food hygiene rating acceptable
- [ ] Kitchen passes visual inspection
- [ ] Packaging is appropriate
- [ ] Meals photographed professionally
- [ ] Calorie information provided
- [ ] Average rating potential: 4.0+

### Nice to Have
- [ ] Organic/healthy certifications
- [ ] Unique cuisine or concept
- [ ] Strong social media presence
- [ ] Existing delivery experience
- [ ] Flexible with menu changes
- [ ] Enthusiastic about partnership

### Deal Breakers
- [ ] Poor hygiene standards
- [ ] Consistent negative reviews
- [ ] Unreliable operations
- [ ] Unwilling to meet standards
- [ ] Location too far
- [ ] Food doesn't match healthy focus

## Frequently Asked Questions (For Partners)

**Q: How do we get paid?**
A: Weekly automatic bank transfers every Monday for previous week's meals.

**Q: What if we can't fulfill an order?**
A: Contact us immediately via WhatsApp. We'll coordinate with customer.

**Q: Can we set our own prices?**
A: No, all meals are included in subscription. You receive fixed payout per meal.

**Q: What happens if a customer complains?**
A: We'll investigate. If valid, we may issue refund. Repeated issues may affect partnership.

**Q: How do we pause orders if we're busy?**
A: Toggle "Accepting Orders" in your dashboard or contact us.

**Q: Can we promote our own restaurant?**
A: Yes! QR codes at your location help drive signups.

**Q: What's the minimum commitment?**
A: No minimum, but we expect consistent availability. 30-day notice to leave platform.
