# تصميم إضافة حفظ مخالفات المركبات PDF

## الهدف

إنشاء إضافة Chrome محلية تستخدم جلسة المستخدم الحالية في موقع وزارة الداخلية، وتفتح تفاصيل المركبات وتحفظ كل صفحة PDF باسم رقم المركبة بعد حذف `00` من بدايته. بعد نجاح اختبار المركبة الأولى، يعالج الإصدار الكامل جميع صفحات المركبات بالتتابع.

## البنية

- `background.js`: يجمع المركبات من جميع صفحات القائمة، يفتحها بالتتابع، ينتظر اكتمال التفاصيل، ويتصل بالتبويب عبر `chrome.debugger` لإنشاء ملفات PDF باستخدام `Page.printToPDF`.
- `dashboard.html` و`dashboard.js`: يطلبان من المستخدم اختيار مجلد الحفظ، يكتبان كل ملف فور إنشائه باستخدام File System Access API، ويعرضان التقدم والنتائج.
- `manifest.json`: يقصر الوصول إلى `https://fees2.moi.gov.qa/*` ويطلب صلاحيات `debugger` و`scripting` و`tabs` فقط.

## تدفق الإصدار الكامل

1. يفتح المستخدم صفحة `companyPlateCount` ويضغط أيقونة الإضافة.
2. تفتح لوحة التحكم وتحفظ معرّف تبويب الوزارة.
3. يختار المستخدم مجلد `المخالفات المرورية` من سطح المكتب.
4. تجمع الإضافة أرقام المركبات من جميع صفحات النتائج وتتحقق من عدم وجود تكرار.
5. لكل مركبة، تعود إلى صفحة القائمة الصحيحة وتضغط رابط التفاصيل.
6. تنتظر صفحة `companyPlateNo` وظهور رقم المركبة وجدول المخالفات.
7. تنشئ PDF عموديًا بحجم A4 الدقيق `210 × 297 مم`، بمقياس محتوى `100%` وخلفيات الصفحة مفعّلة.
8. تحذف `00` من بداية رقم المركبة وتكتب الملف فورًا. إذا كان الملف موجودًا، تتخطاه دون استبدال.
9. تعود للقائمة وتتابع المركبة التالية، مع عرض التقدم وإتاحة الإلغاء بعد المركبة الحالية.

## معالجة الأخطاء

تتوقف الإضافة برسالة واضحة إذا تعذر جمع القائمة أو تغيّرت بنية الصفحات الأساسية. أما خطأ مركبة منفردة فيُسجل وتستمر المعالجة متى أمكن. الملفات الموجودة مسبقًا تُتخطى، وتُفصل جلسة التصحيح في كتلة `finally` حتى عند حدوث خطأ أو إلغاء.

## الاستئناف

لا تستبدل الإضافة ملفات موجودة. لذلك يمكن إعادة تشغيلها بعد الإلغاء أو الانقطاع؛ ستتخطى الملفات المكتملة وتتابع المركبات التي لا تملك ملفًا بعد.
