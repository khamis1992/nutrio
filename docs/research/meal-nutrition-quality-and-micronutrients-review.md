# Meal Nutrition Quality and Micronutrients Review

Date: 2026-07-19

Owner: Agent 2

## Scope

Agent 2 makes partner meal nutrition trustworthy enough for ordering, logging, ranking, and AI reports. The central rule is simple: missing nutrient data must stay missing. It must not be shown, scored, or snapshotted as a measured zero.

## Source Notes

- Open Food Facts uses a nutrient taxonomy with stable IDs and explicit unit metadata. Nutrio follows the same idea locally by storing canonical nutrient codes and canonical units before calculating completeness.
- OpenNutriTracker presents micronutrients in day/week views with reference bars. Nutrio follows that pattern with a daily/weekly adequacy RPC and leaves medical interpretation out of the core calculation.

## Nutrio Decisions

1. Partner meals keep calories and macros as required fields.
2. Fiber, sugar, sodium, and extended micronutrients are nullable unless measured or explicitly entered.
3. Each meal stores:
   - `nutrition_version`
   - `nutrition_provenance`
   - `nutrient_completeness_score`
   - `nutrient_missing_codes`
   - `nutrient_invalid_codes`
4. Completeness is calculated from required macros plus key quality nutrients: fiber, sugar, and sodium.
5. Customer adequacy reads logged meal facts from `meal_history` and returns measured/missing counts for each nutrient.

## Acceptance Mapping

| Requirement | Implementation |
| --- | --- |
| No fake zero is shown as measured data | TypeScript normalization and DB nullable micronutrients preserve `null`. |
| Missing differs from zero | Unit tests cover `null` versus explicit `0`. |
| Snapshots retain historical values | Snapshot helper stores version, provenance, missing codes, and values. |
| Units normalize correctly | Unit tests cover g-to-mg and invalid conversions. |
| Daily/weekly adequacy | `get_user_micronutrient_adequacy(start, end)` aggregates `meal_history`. |
| Partner/admin missing-data queue | `partner_meal_nutrition_missing_queue` lists incomplete or invalid meals. |

## Rollout Notes

The migration is additive. Generated Supabase types should be regenerated by Agent 0 after migrations are frozen. Partner UI can then surface `nutrient_completeness_score` and the missing queue without inventing client-only rules.

## UI Completion

- Partner editors can enter sugar and sodium, identify the source, and leave genuinely unmeasured values empty.
- Admins have a dedicated review queue and can return incomplete meals for correction while hiding them from ordering.
- Customers can switch between daily and seven-day adequacy without treating missing measurements as zero.
- English and Arabic labels are included in the customer card; interpretation remains non-medical.
