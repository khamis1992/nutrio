    Architecture

    Analysis Date: 2025-02-14

    Pattern Overview

    Overall: Client-side SPA with Context-based state management

    Key Characteristics:

    • Multi-role application (Customer, Partner, Admin portals)

    Layers

    Presentation Layer (UI):

    • Used by: End users

    Business Logic Layer:

    • Used by: Presentation layer

    Data Layer:

    • Used by: Business logic layer

    Data Flow

    Authentication Flow:

  • User initiates auth action (sign in/up/out)

  • AuthContext handles action via Supabase client

  • Supabase auth state changes trigger onAuthStateChange callback

  • Context updates user/session state

  • ProtectedRoute guards redirect or render based on auth state

  • Components access auth state via useAuth() hook

    • Query Flow (TanStack Query):

  • Component calls custom hook (e.g., useProfile)

  • Hook initiates Supabase query

  • TanStack Query caches response

  • Component receives data, loading, error states

  • Background refetch on window focus/reconnect

    • Order Placement Flow:

  • User browses meals (/meals, /restaurants/:id)

  • User selects meal and addons

  • Order data saved to meal_schedules or orders

  • Subscription quota checked/incremented

  • Notification sent to user and partner

  • Real-time UI updates

    • State Management:

    • Form state: React Hook Form

    Custom Hooks Pattern:

    • Used in: Page components and shared components

    UI Component Pattern:

    • Used in: All page and feature components

    • Used in: src/App.tsx for protected routes
